下面这套可以直接给 **Trae 自定义 Agent / SOLO Agent** 使用。核心目标是：**让 Trae 周期性分析项目、拆分任务、持续实现功能，但禁止部署、禁止真实环境测试、禁止高权限命令，最终按“企业级可商用 AIOps 智能运维平台”标准推进。**

Trae 官方文档里支持自定义 Agent，并可配置 Prompt、MCP、内置工具；Skills 也可以作为“专业能力文档/操作规范”提供给 Agent 使用。([TRAE][1]) 但我没有看到官方明确提供类似 `cron` 的稳定“按时间自动触发”能力，所以建议你把下面内容做成 **固定 Agent Prompt + 每次触发任务 Prompt**，定时部分交给 Trae 自身任务入口、飞书机器人、系统定时器或你手动每天触发一次。

工作方式：
每一轮任务都必须按以下流程执行：

第一步：项目现状扫描

- 自动化运维：Playbook、审批流、执行记录、回滚建议、操作审计。
- AI 助手：基于告警、日志、指标、知识库、Runbook 的智能问答、故障分析、处置建议。
- 知识库：故障案例、SOP、应急预案、处理手册、复盘报告。
- CMDB / 资产管理：主机、服务、应用、集群、负责人、环境、标签、依赖关系。

第四步：实现要求

- 优先复用已有目录结构、组件、接口风格、状态管理方式、UI 风格。

- 新增代码要可读、可维护、可扩展，符合企业级项目结构。
- 所有新增能力都要考虑后续接入真实数据源的扩展点。

第五步：质量约束

第六步：每轮输出固定报告

# 四、企业级 AIOps 对标标准

你可以让 Trae 按这个标准长期推进。

主流 AIOps 平台的共同趋势是：**统一遥测数据、告警降噪、异常检测、根因分析、事件协同、自动化处置和 AI 辅助排障**。例如 Dynatrace 的 RCA 会基于拓扑和上下文自动定位复杂问题的根因；New Relic 强调跨日志、指标、链路、变更的关联分析和 AIOps 事件管理；PagerDuty AIOps 聚焦告警降噪、事件可见性和高效分诊；Datadog Watchdog 覆盖异常检测、自动 RCA 和影响分析。([docs.dynatrace.com][2])

所以你的项目不要一开始追求“AI 很炫”，而是要按下面顺序做。

| 阶段      | 目标       | 重点功能                         |
| ------- | -------- | ---------------------------- |
| 第 1 阶段  | 平台骨架成型   | 首页大屏、菜单、路由、权限框架、基础布局         |
| 第 2 阶段  | 运维核心数据   | 资产 CMDB、服务管理、环境管理、标签体系       |
| 第 3 阶段  | 告警中心     | 告警列表、等级、来源、去重、聚合、抑制、状态流转     |
| 第 4 阶段  | 事件中心     | 事件创建、认领、升级、处理记录、恢复、复盘        |
| 第 5 阶段  | 巡检中心     | 主机/K8s/数据库/中间件巡检、风险评分、报告生成   |
| 第 6 阶段  | RCA 根因分析 | 拓扑、时间线、变更关联、指标日志链路联合分析       |
| 第 7 阶段  | AI 助手    | 告警解释、日志摘要、故障分析、Runbook 推荐    |
| 第 8 阶段  | 自动化运维    | Playbook、审批、执行记录、回滚建议        |
| 第 9 阶段  | 商用能力     | 多租户、审计日志、License、插件市场、系统设置   |
| 第 10 阶段 | 企业级体验    | SLO、MTTR、MTTD、报表、通知集成、开放 API |

# 十、最终建议

你的 Trae 提示词不要写成：

```text
帮我持续优化这个项目，做到企业级。
```

这种太虚，Agent 容易乱改。

应该写成：

```text
你每轮只做一个垂直切片；
你必须维护路线图和任务池；
你不能部署测试；
你不能执行高风险命令；
你必须对标 AIOps 商用平台；
你必须每轮输出变更报告；
你必须让项目逐步形成资产、告警、事件、巡检、RCA、AI 分析、自动化、权限审计的完整闭环。
```

最适合你当前使用的完整策略是：

```text
Trae 自定义 Agent Prompt
+ docs/NEXT_TASKS.md 任务池
+ docs/AI_AGENT_WORKLOG.md 工作日志
+ 每天/每轮固定触发 Prompt
+ 禁止部署/高权限命令
+ 每轮只做一个 AIOps 垂直功能
```

这样它才能长期稳定推进，而不是每次都重新分析、重复建议、随机重构。
