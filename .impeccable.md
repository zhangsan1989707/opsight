# Design Context — Opsight

## Target Audience
政府/企业 IT 运维团队（运维工程师、SRE、IT 管理员）。
使用场景：监控服务器集群健康状态、处理故障告警事件、分析指标趋势。
工作环境：办公室工作站（1920×1080 为主）、值班室大屏、偶尔移动端查阅。

## Brand Personality
- **可信赖** — 政府级产品，安全可靠是最重要的感知
- **专业但不冷酷** — 运维工具不需要花哨，但不能冰冷到让人不想用
- **高效清晰** — 信息密度高但组织有序，一眼看到问题所在
- **适度现代** — 不能像 2010 年的传统网管软件，但也不需要过于炫技

## Tone
专业监控仪表盘风格，偏向 Datadog/Grafana 路线但要有自己的品牌辨识度。
必须支持亮色和暗色双主题（政府场景亮色需求高）。
避免 AI 生成暗色仪表盘的常见套路（近纯黑背景、单发光强调色、毛玻璃滥用）。

## Constraints
- Next.js 14 + React 18 + Tailwind CSS — 不可更换技术栈
- Chart.js 已集成 — 图表库保持不变
- 字体已离线 (Space Grotesk + JetBrains Mono) — 可替换但不强制
- 需支持离线部署（无 CDN 依赖）
- 需适配 1920×1080 和 1366×768 两种主流分辨率
- 等保合规：必须有审计日志页面入口

## Differentiation Point
"可信赖的现代运维仪表盘" — 不像竞品那样冷冰冰的暗色终端风，而是有温度、有层次、让人愿意长时间盯屏的专业工具。
